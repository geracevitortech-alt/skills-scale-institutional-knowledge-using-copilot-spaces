## Change Control Process Reference

### When to Submit a Change Request
Submit a change request when:
- Adding or removing project scope or deliverables
- Modifying project timeline or major milestones
- Changing resource allocation or budget
- Altering technical approach or architecture
- Updating requirements or acceptance criteria
- Any change that impacts the approved project plan

### Change Control Workflow
1. **Submit:** Requestor completes this template and submits to Change Control Lead
2. **Review:** Change Control Lead reviews for completeness and impact analysis
3. **Assess:** Project team assesses technical, schedule, and resource impacts
4. **Board Review:** Change Control Board meets to evaluate and decide
5. **Communicate:** Decision and rationale communicated to all stakeholders
6. **Implement:** If approved, change is implemented per the plan
7. **Track:** Monitor implementation and document actual outcomes
8. **Close:** Post-implementation review and lessons learned

### Change Priority Guidelines
- **Critical:** Addresses blocking issue, security vulnerability, or compliance requirement
- **High:** Significantly improves project outcomes or customer value
- **Medium:** Moderate improvement or addresses non-blocking issue
- **Low:** Nice-to-have enhancement or minor adjustment
